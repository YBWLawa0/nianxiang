# 念想（NianXiang）项目部署方式

本文档说明如何在本地与生产环境部署本仓库（Vue 3 + Vite 前端、FastAPI + MySQL 后端）。移动端（Capacitor）见文末简要说明。

---

## 一、架构与端口约定

| 组件 | 说明 |
|------|------|
| 前端 | `frontend/`，开发默认 `http://localhost:5173`（Vite） |
| 后端 | `backend/`，API 根路径含 `/api`，健康检查 `GET /api/health` |
| 数据库 | MySQL，连接串见后端 `.env` 中 `DATABASE_URL` |

前端通过环境变量 `VITE_API_BASE_URL` 指向后端 API 根路径（默认开发为 `http://localhost:8000/api`，见 `frontend/src/api/client.ts`）。

---

## 二、部署前准备

1. **MySQL**：创建数据库（示例库名 `nianxiang`，字符集 `utf8mb4`），并确保账号可从运行后端的机器访问。
2. **Python**：建议 3.11+（与团队环境一致即可）。
3. **Node.js**：建议当前 LTS，用于构建前端；Capacitor 打包另需 Android Studio / Xcode。

---

## 三、后端部署（FastAPI）

### 1. 安装依赖

```bash
cd backend
python -m venv .venv
# Windows: .venv\Scripts\activate
# Linux/macOS: source .venv/bin/activate
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `backend/.env.example` 为 `backend/.env`，按环境修改：

| 变量 | 说明 |
|------|------|
| `SECRET_KEY` | 生产环境务必改为足够长的随机字符串 |
| `DATABASE_URL` | MySQL 连接，格式见示例文件 |
| `DASHSCOPE_API_KEY`、`QWEN_MODEL` | 千问 / DashScope 相关能力 |
| `FRONTEND_ORIGINS` | 允许跨域的前端源，**逗号分隔**。生产需加入实际站点，例如 `https://your-domain.com` |

### 3. 启动服务

开发常用：

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

生产建议：

- 使用 **systemd / supervisord** 或容器托管上述命令（去掉 `--reload`）。
- 前面可挂 **Nginx** 反代到 `127.0.0.1:8000`，并配置 HTTPS。

应用启动时会根据模型创建表（`Base.metadata.create_all`）。若你后续引入 Alembic 迁移，以迁移流程为准。

## 四、前端部署（静态资源 + API 地址）

### 1. 安装与构建

```bash
cd frontend
npm install
```

生产构建前设置 API 地址（与实际上线域名一致）：

```bash
# 示例：后端 API 对外为 https://api.example.com/api
set VITE_API_BASE_URL=https://api.example.com/api
npm run build
```

Linux/macOS 使用 `export VITE_API_BASE_URL=...`。

构建产物在 `frontend/dist/`，可部署到任意静态托管（Nginx、OSS+CDN、Netlify 等）。

### 2. 与后端的配合

- 浏览器访问的前端域名必须出现在后端 `FRONTEND_ORIGINS` 中，否则会出现 CORS 错误。
- `VITE_API_BASE_URL` 应指向**浏览器可访问**的 API 根路径（含 `/api` 后缀，与代码默认一致）。

### 3. 本地预览构建结果（可选）

```bash
cd frontend
npm run preview
```

---

## 五、典型生产拓扑（Nginx 示例思路）

1. **前端**：`root` 指向 `dist`，`try_files` 回退到 `index.html`（Vue Router  history 模式若启用时需如此）。
2. **后端**：`location /api`（或整站反代）`proxy_pass` 到 Uvicorn。
3. 全站 HTTPS；`FRONTEND_ORIGINS` 写前端页面的 `https` 源。

具体 Nginx 片段需按你的域名与路径微调，此处不展开固定配置。

## 六、移动端（Capacitor，可选）

项目已集成 Capacitor（`frontend/capacitor.config.ts`，`webDir` 为 `dist`）。

常规流程：先 `npm run build` 生成 `dist`，再执行 `npx cap sync` / `npx cap open android`（或 iOS）按官方文档生成原生工程并打包。**移动端访问的 API 基地址**同样需在构建前通过 `VITE_API_BASE_URL` 指向可公网访问的后端；若使用 HTTPS 自签名证书，需在原生侧配置信任或使用合法证书。

---

## 七、健康检查与排错

- 后端：`GET /api/health`，应返回 `{"status":"ok"}`。
- 前端空白或接口失败：检查 `VITE_API_BASE_URL` 是否重建、浏览器控制台网络请求与 CORS 报错、`FRONTEND_ORIGINS` 是否包含当前页面源。
- 数据库：确认 `DATABASE_URL` 与 MySQL 用户权限、防火墙、库是否已创建。
